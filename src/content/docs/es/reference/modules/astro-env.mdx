La API `astro:env` te permite configurar un esquema tipado de forma segura para las variables de entorno que hayas definido. Esto te permite indicar si deben estar disponibles en el servidor o en el cliente, así como definir su tipo de dato y propiedades adicionales. Para ejemplos e instrucciones de uso, [consulta la guía de `astro:env`](/es/guides/environment-variables/#type-safe-environment-variables).

## Importaciones desde `astro:env`

```js
import { getSecret } from "astro:env/server";
```

### `getSecret()`

<p>
<Since v="5.0.0" />
</p>

La función auxiliar `getSecret()` permite obtener el valor sin procesar de una variable de entorno por su clave.

Por ejemplo, puedes obtener un valor booleano como cadena:

```js
import {
  FEATURE_FLAG, // booleano
  getSecret,
} from "astro:env/server";

getSecret("FEATURE_FLAG"); // string | undefined
```

Esto también puede ser útil para obtener un secreto que no esté definido en tu esquema, por ejemplo uno que dependa de datos dinámicos de una base de datos o API.

Si necesitas obtener variables de entorno de forma programática, recomendamos usar `getSecret()` en lugar de `process.env` (o equivalente). Como su implementación la proporciona tu adaptador, no tendrás que actualizar todas tus llamadas si cambias de adaptador. Por defecto, utiliza `process.env` en desarrollo y build.
